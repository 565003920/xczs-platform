## 三、v1.5 — 增强分析（4周）

> 目标：让诊断结论从"描述性"升级为"证据性"

### Sprint 1: 对比分析引擎（第1-2周）

| ID | 功能 | 描述 | 前端 | 后端 | 优先级 |
|----|------|------|:----:|:----:|:------:|
| **F24** | 模式 A/B 对比 | 同一课程不同学期的模式效果对比：成绩分布、满意度、留存率 | 🖥️ | 🐍 | P0 |
| **F25** | 跨教师模式对标 | 同课程不同教师的模式横向对比，识别高效能模式 | 🖥️ | 🐍 | P0 |
| F15 | 年级/专业横向对比 | 同专业不同班级学情对比，识别异常班级 | 🖥️ | 🐍 | P1 |
| F17 | 教师教学效能趋势 | 教师历学期模式变化 + 成绩变化 + 评教变化三线图 | 🖥️ | 🐍 | P1 |

**后端新增**：
- `app/services/comparison.py` — A/B 对比计算（t检验 + Cohen's d 效应量）
- `app/routers/comparison_routes.py` — 3 个新接口
- 数据库：新增 `semester_index` 字段支持时间序列对比

**前端新增**：
- `src/pages/ComparisonAB.tsx` — A/B 双雷达图 + 效应量卡片
- `src/pages/TeacherTrend.tsx` — 三线趋势图

### Sprint 2: 教学模式库（第3-4周）

## 八、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:----:|:----:|----------|
| 真实数据获取困难 | 中 | 高 | v1.5 阶段仍使用模拟数据；提前联系合作院校 |
| LLM API 不稳定/成本高 | 中 | 中 | 核心分析用规则引擎，LLM 仅做文本增强（可降级） |
| 团队人力不足 | 高 | 高 | 严格落实优先级：P0 > P1 > P2；v1.5 砍掉 P2 功能 |
| 竞赛截止时间提前 | 低 | 高 | 每个版本都有可演示状态，不依赖完整交付 |
| 校际数据共享合规 | 中 | 中 | v3.0 前完成数据合规审查；默认脱敏 |
